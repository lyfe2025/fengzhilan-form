## 使用场景

以下几种情况，可以把这个表单链接发给客户：

1. **初次咨询**：客户通过各渠道（朋友介绍、广告、社交媒体）找到我们，想了解做软件大概多少钱、多长时间。直接发表单链接，比来回聊天问需求高效得多。

2. **朋友圈/社群获客**：发朋友圈或在行业群做软件定制科普时，附上表单链接，感兴趣的人可以直接填写，省去一对一聊天的人力成本。

3. **老客户转介绍**：老客户帮忙推荐时，让他把表单转给对方。表单自带知识引导页，帮对方先建立基本认知（软件类型、费用范围、合作须知），减少我们前期重复解释的工作。

4. **需求不够清晰的客户**：有些客户只有模糊的想法（"我想做个小程序"），但说不出来具体要什么。表单里的引导和提示词可以帮他们梳理思路，填完后我们拿到的信息质量也更高。

5. **居间人/中间人转发**：有朋友或合作伙伴说"我认识一个人有开发需求"，直接把表单链接发给中间人让他转发。表单自带科普引导，中间人不需要帮忙解释技术问题，客户填完后我们直接对接，降低中间人的沟通负担。

6. **筛选预算匹配的客户**：表单第一题就问预算，预算低于 2 万会给出提示。这样可以提前筛选，把精力放在真正能合作的客户身上。

## 快速开始

### 本地预览

这是一个纯静态页面，无需安装任何依赖。直接用浏览器打开即可：

```bash
# macOS
open index.html

# 或者用 VS Code 的 Live Server 插件预览
# 或者用任意 HTTP 服务器
npx serve .
```

### 部署

因为只有一个 `index.html` 文件，部署到任意静态托管平台即可：

- **GitHub Pages**：推到仓库后在 Settings → Pages 中开启
- **Vercel / Netlify**：直接导入仓库，零配置部署
- **任意服务器**：把 `index.html` 放到 Nginx/Apache 的静态目录下

## 页面结构说明

整个页面分为三屏，按顺序展示：

### 第一屏：知识引导页

客户打开链接后最先看到的页面。目的是**在填表前先帮客户建立认知**，避免出现"我要做个 App，预算 2000"这类预期错位的情况。采用手风琴折叠设计，默认全部收起，首屏即可看到所有标题和 CTA 按钮。包含三个可展开板块 + 一个始终可见的合作原则：

- **软件类型科普**（可展开）：小程序、App、PC 后台、H5 的区别和适用场景
- **费用与周期参考**（可展开）：让客户对行情有基本概念
- **合作须知**（可展开）：上线后的持续费用、需要客户配合、代码归属等
- **合作原则**（始终可见）：明确只接有预算的付费项目，过滤掉不靠谱的需求

客户阅读后点击「我已了解，开始填写需求」进入下一屏。

### 第二屏：表单页

7 道题，3 道必填（预算、需求描述、联系方式），4 道选填。设计逻辑：

- **预算前置**（Q1）：第一题就问预算，快速判断是否匹配
- **自由描述**（Q2）：用开放式文本框代替功能勾选，让客户用自己的话说需求，信息更真实
- **参考产品**（Q3）：帮我们快速理解客户的期望标准
- **平台选择**（Q4）：多选，配合引导页的科普，客户此时已有基本认知
- **时间预期**（Q5）：了解紧急程度
- **补充说明**（Q6）：兜底字段，特殊需求都放这里
- **联系方式**（Q7）：姓名 + 微信号必填，手机号选填

### 第三屏：成功页

提交后展示，告知客户"1 个工作日内联系"，同时展示微信号卡片，支持点击一键复制。
